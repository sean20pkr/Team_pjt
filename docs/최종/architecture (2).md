# architecture.md — 입력 → 해석 → 전망 → 보고초안

> `PRD.md` 기준의 MVP 동작 구조 요약. 첫 버전은 `월 단위 조회·설명`과 `전사 총계 전망 작성 지원`에 집중한다.

| 섹션 | 내용 |
|---|---|
| Goal | 채널 물량 관련 질문에 대해 `조회·설명 + 전사 총계 전망 + 경쟁보험사 비교 + 채널별 방향성 + 보고용 초안`을 제공한다 |
| Input | 사용자 질문(텍스트) · `monthly_summary`(월별 총괄 요약) · `main_fact`(채널·상품 상세 팩트) · `monthly_events`(월별 이벤트/이슈) · `special_products`(신상품/중점상품) · `channel_profile` · `product_profile` · 자유 메모 3~5줄 · 회의자료 업로드 후 요약 |
| Input Rule | 첫 버전은 비식별 또는 집계 데이터 우선 · 채널 이슈 입력 최소 항목은 `이슈 내용 / 적용 채널 / 기간 / 확신도` · 개인 식별정보와 상세 민감정보는 제외 · `monthly_summary`와 `main_fact`는 합계 검산 규칙을 먼저 통과해야 한다 |
| Process | ① 질의 유형 분류(조회형/설명형/예측형) → ② `monthly_summary`에서 총계 기준값과 경쟁보험사 주요 지표 조회 → ③ `main_fact`에서 채널·상품 기여분 조회 → ④ `monthly_events + special_products + 자유 메모`에서 이슈, 신상품, 경쟁보험사 특이사항 근거 연결 → ⑤ 최신성·누락·합계 검산 점검 → ⑥ 전사 총계 조회값 또는 전망 범위 산출 → ⑦ 경쟁보험사 대비 관점 정리 → ⑧ 채널별 방향성 생성 → ⑨ 상위 3개 영향 요인과 흔들림 조건 정리 → ⑩ `markdown` 기준 보고용 문장 초안 구성 |
| Logic | 출력 순서는 `전사 총계 먼저 → 경쟁보험사 비교 → 채널별 방향성 → 상위 3개 영향 요인 → 흔들림 조건 → 보고 초안`으로 설계한다 |
| Tools | Supabase(`monthly_summary / main_fact / monthly_events / special_products / reference tables` 저장·조회) · BizRouter LLM(설명·전망·보고 문장 생성) · (선택)질의 분류기(조회형/설명형/예측형 라우팅) · 검산 로직(합계·무결성 체크) · 엑셀/회의자료 업로드 파이프라인 |
| Output | 전사 총계 전망 범위와 중심값 · 경쟁보험사 대비 변화 또는 비교 관점 · 채널별 증가/보합/감소 방향성 · 상위 3개 영향 요인 · 전망이 상단/하단으로 흔들릴 수 있는 조건 · 보고용 서술 초안 |
| Output Format | MVP 기본 형식은 `markdown` · 후속 확장 대상은 `json/csv/report` |
| HITL | 데이터 최신성은 운영 담당자가 확인 · 총계 전망과 채널별 방향성은 실무자가 검토 · 경영진 제출용 문안은 의사결정자 또는 보고 책임자가 최종 승인 |
| Failure | 데이터 최신화 누락 시 전망 중단 또는 경고 · 근거 부족 시 단정 금지 및 `추가 확인 필요` 표시 · 변수 충돌 시 복수 가능성 또는 흔들림 조건으로 제시 |
| Out of Scope | 주별/일별 실적 분석 · 임원 보고 직전 사전 점검 로직 · `json/csv/report` 본격 지원 · 채널/상품 단위 정밀 예측 · 자동 배포 |
| Cost | 기준 가정은 `파일럿 5~8명`, 월 사용량 `조회형 1,000회 / 예측·설명형 250회 / 임원보고 초안 40회 / 문서 적재·요약 80건`, 그리고 `검색/RAG + 요약 + 캐시` 구조 전제다. 작업별 토큰은 조회형 `입력 8k~10k / 출력 700~1,000`, 예측·설명형 `입력 15k~20k / 출력 1.5k~2k`, 임원보고 초안 `입력 25k~35k / 출력 2k~3k`, 문서 적재·요약 `입력 10k~15k / 출력 500~1,000`으로 본다. 기준 파일럿 총량은 `입력 약 1,466만 / 출력 약 168만 / 총 1,634만 토큰/월` 수준이며, 모델별 월 토큰 비용은 `GPT-5.4 mini 약 $18.6 / GPT-5.4 약 $61.9 / GPT-5.5 약 $123.8`으로 추정한다. 8개 부서 16명 파일럿은 `mini $37.2 / GPT-5.4 $123.8 / GPT-5.5 $247.6` 수준, 150명 확산은 `mini $348.8 / GPT-5.4 $1,160.6 / GPT-5.5 $2,321.3` 수준으로 본다. 다만 이 수치는 토큰/API 비용만 반영한 값이며 저장소, 검색 인프라, 모니터링, 보안 연동 비용은 별도다. 실제 비용의 핵심 변수는 모델 자체보다 `문서를 매번 얼마나 길게 넣느냐`이므로, 검색·요약·캐시 구조를 전제로 설계해야 한다 |

## 상위 3개 영향 요인 후보군 정의
- `추세 요인`: 최근 1~2개월 추세, 전월/전년 대비 변화, 직전 반등·둔화 패턴
- `상품·시책 요인`: 신상품 출시, 중점상품 운영, 시책 전개, 프로모션 강도 변화
- `영업 캘린더 요인`: 영업일수, 월초/월말 구조, 계절성, 휴일 효과
- `채널 현장 요인`: 채널별 대내외 이슈, 운영 정책 변화, 자유 메모 기반 현장 특이사항
- `시장·경쟁 요인`: 시장 규모 변화, 경쟁보험사 프로모션, 경쟁사 상품 출시, 상대 점유율 압박
- 선정 규칙: 모델은 후보군 전체를 검토한 뒤 해당 월 총계 전망과 채널 방향성에 가장 설명력이 큰 3개만 우선순위로 제시한다.
- 출력 원칙: `상위 3개 영향 요인`은 고정 템플릿이 아니라 질의 시점별 동적 선정 항목이며, 필요 시 실무자가 순서를 조정하거나 교체할 수 있다.
