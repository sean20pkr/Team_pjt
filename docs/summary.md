# 문서 요약표

## 1. 문서 한눈에 보기

| 문서 | 역할 | 핵심 질문 | 현재 담은 내용 | 다음 보완 포인트 |
| --- | --- | --- | --- | --- |
| `prd.md` | 무엇을 왜 만들지 정리 | 이 에이전트는 어떤 문제를 해결하고, 무엇을 성공으로 볼 것인가 | 목표, 비목표, 요구사항, 성공률 초안, 리스크 | 성공률 목표값, 승인 기준 수치화 |
| `customer.md` | 누가 왜 쓰는지 정리 | 사용자는 무엇을 기대하고 무엇을 걱정하는가 | 고객군, 기대 결과, 불편, 반대 근거, 검증 질문 | 신뢰 조건, 실제 사용 맥락별 세분화 |
| `architecture.md` | 어떻게 동작하는지 정리 | 요청이 어떻게 판단되고 실행되며 검증되는가 | 구조, 실행 흐름, 승인 구조 초안, 실패 처리 | 입력/출력 기준, 로그 형식, 재시도 규칙 |
| `AGENTS.md` | 에이전트 운영 규칙 정리 | 에이전트는 어떤 원칙으로 일해야 하는가 | 기본 원칙, 승인 원칙, 방지 규칙, 완료 보고 규칙 | 중단 조건, 승인 예외 규칙, 평가 루틴 |

## 2. 문서 간 연결 관계

| 시작 문서 | 연결 문서 | 연결 이유 |
| --- | --- | --- |
| `prd.md` | `customer.md` | 목표가 실제 사용자 문제와 맞는지 확인하기 위해 |
| `prd.md` | `architecture.md` | 요구사항이 실행 구조로 이어지는지 확인하기 위해 |
| `architecture.md` | `AGENTS.md` | 구조상 필요한 판단 규칙을 운영 원칙으로 고정하기 위해 |
| `customer.md` | `AGENTS.md` | 사용자의 기대와 불안을 실제 행동 원칙에 반영하기 위해 |

| 실패 대응 | 방향만 있음 | 재시도, 보류, 종료 기준 확정 |

## 4. GitHub harness 반영 점검

- 참고 링크: [channel-volume-agent-docs / harness](https://github.com/ktheee8181-png/channel-volume-agent-docs/tree/main/harness)
- 현재 문서에는 `목표`, `고객`, `구조`, `운영 원칙` 같은 상위 기획 요소는 반영되어 있습니다.
- 다만 현재 문서는 아직 `실행용 하네스` 수준까지는 충분하지 않습니다.
- 특히 아래 항목은 추가되면 더 좋아집니다.

| 점검 항목 | 현재 반영 수준 | 보완 의견 |
| --- | --- | --- |
| 실행 시나리오 모음 | 낮음 | 대표 작업 5~10개를 표준 시나리오로 정리 |
| 성공/실패 판정 기준 | 중간 | 성공률 외에 실패 유형 분류표 추가 |
| 승인 기준표 | 낮음 | 위험도별 승인 여부를 표로 고정 |
| 입력/출력 계약 | 낮음 | 어떤 입력을 받고 어떤 결과를 내는지 형식 정의 |
| 평가용 테스트셋 | 낮음 | 더미 작업 예시와 정답 기준 필요 |
| 운영 로그 기준 | 중간 | 어떤 로그를 남길지 필드 수준으로 정의 |
| 예외 처리 룰 | 중간 | 재시도, 중단, 사람 호출 조건을 더 구체화 |

## 5. 한 줄 결론

- 지금 4개 문서는 `기획 초안`으로는 충분히 출발 가능합니다.
- 하지만 첨부하신 `harness` 수준에 더 가까워지려면, 다음 단계에서 `승인 기준표`, `대표 시나리오`, `평가 기준표`를 추가하는 것이 가장 중요합니다.
