# PRD

## 1. 문서 목적
- 이 문서는 `채널물량 인사이트 AI AGENT`가 어떤 문제를 해결하고, 어떤 기능을 제공하며, 무엇을 성공으로 볼지 정리한 제품 요구사항 문서입니다.
- 기획, 구현, 테스트, 운영 검토의 공통 기준으로 사용합니다.

## 2. 서비스 한 줄 정의
- 이 서비스는 채널물량 관련 과거 데이터를 빠르게 검색하고, 원하는 시점의 물량을 예측하며, 타겟 물량 달성을 위한 인사이트까지 제공하는 보조형 AI 서비스입니다.

## 3. 문제 정의
### 3-1. 현재 문제
- 필요한 숫자와 문서가 여러 파일과 자료에 흩어져 있습니다.
- 특정 월의 물량 변화 원인을 빠르게 설명하기 어렵습니다.
- 경쟁보험사 대비 우세/열세를 같은 기준으로 보기 어렵습니다.
- 다음달 물량을 예측할 때 숫자와 근거를 함께 정리하는 데 시간이 많이 듭니다.
- 보고자료를 만들 때 사람 기억과 경험에 대한 의존도가 높습니다.

### 3-2. AS-IS
- 담당자가 관련 자료를 여러 곳에서 따로 찾습니다.
- 엑셀, 회의자료, 시책 문서, 과거 메모를 각각 확인합니다.
- 맥락이 부족하면 인적 데이터 기반으로 추가 확인합니다.
- 담당자가 숫자와 배경을 다시 정리해 내부 판단 자료나 보고자료를 만듭니다.

### 3-3. TO-BE
- 담당자가 먼저 AGENT에 질문합니다.
- AGENT가 과거 물량, 이슈, 시책, 경쟁보험사 흐름을 함께 보여줍니다.
- AGENT가 다음달 총계 숫자와 채널별 방향성, 근거를 함께 제시합니다.
- 사람은 검토, 수정, 보고에 집중합니다.

## 4. 목표
- 월별 물량과 관련 맥락을 빠르게 조회할 수 있게 합니다.
- 사용자가 원하는 시점의 물량을 참고용으로 예측합니다.
- 타겟 물량 달성에 도움이 되는 인사이트를 제공합니다.
- 보고자료 작성을 더 빠르고 일관되게 돕습니다.

## 5. 비목표
- AI가 공식 보고를 최종 확정하지 않습니다.
- AI가 사람 승인 없이 경영 판단을 대신하지 않습니다.
- 사람이 사전에 시책효과 강도를 억지로 수치화하도록 요구하지 않습니다.
- 실제 고객정보, 계약정보, 사내 기밀정보는 사용하지 않습니다.

## 6. 핵심 기능
### 6-1. 빠른 검색 기능
- 과거 특정 시점의 물량, 판촉비, 프로모션, 경쟁사 이슈, 시장 이슈를 빠르게 찾을 수 있어야 합니다.
- 숫자만이 아니라 당시 시책, 신상품, 운영 배경도 함께 보여줘야 합니다.

### 6-2. 예측 기능
- 사용자가 원하는 시점의 물량을 가능한 한 정확하게 예측해야 합니다.
- 총계 기준 숫자와 전월 대비 방향성을 함께 제시해야 합니다.
- 채널별 설명과 경쟁보험사 대비 비교 관점이 같이 나와야 합니다.

### 6-3. 인사이트 제공 기능
- 입력된 데이터와 사용자의 질문을 바탕으로 타겟 물량 달성을 위한 시사점을 제시해야 합니다.
- 과거 상황과 현재 이슈를 함께 읽고, 어떤 채널을 더 봐야 하는지 설명해야 합니다.

### 6-4. 보고 보조 기능
- 사람이 검토 후 바로 다듬을 수 있는 문장 초안을 제공해야 합니다.
- 내부 공유용 요약과 임원 보고용 참고 문장을 구분해 줄 수 있어야 합니다.

## 7. 핵심 사용자
- 월별 물량과 채널 흐름을 확인하는 실무 담당자
- 내부 판단 자료와 보고자료를 준비하는 기획/운영 담당자
- 채널별 흐름과 경쟁 상황을 빠르게 파악해야 하는 유관부서 파트장

## 8. 입력 데이터
### 8-1. 숫자형 입력
- 월별 업적 물량
- 조직 효율 숫자
- CSM
- 시장 규모
- 경쟁보험사 숫자 및 비교 지표
- 판촉비

### 8-2. 문서형 입력
- 월별 시책 세부내용
- 신상품/운영 공지
- 채널별 특이 이슈 메모
- 기존 보고자료
- 프로모션 관련 문서

### 8-3. 운영 입력
- 담당자가 직접 넣는 월별 이슈
- 기존 보고자료 업로드

## 8-4. mock 데이터 세트
- `channel_profile.csv`: 채널 기준 정보 검증용
- `product_profile.csv`: 상품 기준 정보 검증용
- `special_products.csv`: 특이 상품 또는 집중 관리 상품 검증용
- `main_fact.csv`: 검색, 예측, 인사이트의 핵심 수치 검증용 사실 데이터
- `monthly_events.csv`: 월별 시책, 프로모션, 운영 이슈 검증용
- `monthly_summary.csv`: 월별 요약 지표와 집계 검증용
- `channel_volume_dummy_data.xlsx`: 실무자가 보기 쉬운 통합 더미 데이터 검토용
- `dummy_data_bundle.json`: 데이터 묶음 구조와 시스템 연계 검증용

## 9. 입력 원칙
- 사람이 입력할 수 있는 범위만 요구합니다.
- 시책효과 강도처럼 사전에 알 수 없는 값은 사람이 직접 억지로 수치화하지 않습니다.
- AI는 과거 패턴과 동반 변수들을 바탕으로 시책효과를 추정하는 보조 역할을 맡습니다.
- 경쟁보험사 비교는 동일 기준으로 맞춘 데이터만 사용합니다.

## 10. 기대 출력
### 10-1. 검색 출력
- 특정 시점의 핵심 숫자
- 판촉비와 프로모션 정보
- 당시 시책, 이슈, 운영 배경
- 경쟁보험사 대비 비교 설명
- 시장 이슈 요약

### 10-2. 예측 출력
- 특정 시점의 총계 기준 숫자 예측
- 전월 대비 방향성
- 채널별 증가/감소 가능성 설명
- 예측 근거
- 추가 확인 포인트

### 10-3. 인사이트 출력
- 타겟 물량 달성을 위한 핵심 해석
- 채널별 우선 대응 포인트
- 경쟁사 대비 대응 시사점
- 추가 확인이 필요한 위험 신호

### 10-4. 보고 보조 출력
- 내부 공유용 요약 문장
- 임원 보고 참고용 초안 문장
- 체크 포인트 목록
- 1장 요약 자료의 초안 구조

## 10-5. mock 데이터 기준 검증 질문
- 특정 월의 채널별 물량을 바로 찾을 수 있는가
- 특정 월의 판촉비, 프로모션, 월별 이벤트를 함께 묶어 보여줄 수 있는가
- 경쟁보험사 비교 관점을 월별 요약과 함께 설명할 수 있는가
- 원하는 시점의 물량 예측에 필요한 과거 흐름 데이터가 연결되는가
- 타겟 물량 달성을 위한 인사이트를 과거 이벤트와 현재 조건을 함께 묶어 설명할 수 있는가

## 11. 출력 원칙
- 총계 기준 숫자 예측이 가능하면 제시합니다.
- 전월 대비 방향성이 맞아야 합니다.
- 채널별 설명이 논리적으로 따라와야 합니다.
- 근거와 한계를 함께 보여줍니다.
- 데이터가 부족하면 부족하다고 분명히 말합니다.
- 경쟁보험사 숫자 및 상대 우세/열세 비교 관점을 일관되게 포함합니다.
- `월말 착지` 대신 `월말 마감` 표현을 사용합니다.

## 12. 운영 원칙
- AI AGENT는 보조 도구입니다.
- 최종 판단과 공식 보고는 사람이 합니다.
- 예측 숫자는 참고용 인사이트이며 사람 검토가 선행되어야 합니다.
- 설명은 반드시 근거 중심이어야 합니다.

## 13. 성공 기준
### 13-1. 핵심 KPI
- 빠른 검색 성공률
- 예측 응답 성공률
- 인사이트 활용률
- 사람 검토 후 활용된 답변 비율
- 보고 초안 재사용률
- 경쟁보험사 비교 설명 활용률

### 13-2. 성공률 정의 초안
- 에이전트 성공률 = `사람 검토 기준으로 유의미하게 활용 가능한 응답 수 / 전체 응답 수`

### 13-3. mock 데이터 검증 기준
- 검색 기능: mock 데이터에 있는 특정 월, 특정 채널, 특정 이벤트를 정확히 찾아야 합니다.
- 예측 기능: 과거 흐름을 바탕으로 원하는 시점의 참고용 예측을 만들 수 있어야 합니다.
- 인사이트 기능: 숫자만이 아니라 이벤트, 프로모션, 채널 차이까지 함께 설명해야 합니다.
- 보고 보조 기능: mock 데이터 기반으로 사람이 바로 수정 가능한 초안 문장을 제공해야 합니다.

## 15. 구현 우선순위
1. 입력 데이터 템플릿 표준화
2. 조회형 기능에 필요한 데이터 스키마 확정
3. 조회형 위에 예측형을 추가하는 2단계 구조 설계
4. 경쟁보험사 비교 기준 통일
5. 사람 승인 시점과 보고 반영 프로세스 구체화

## 16. 같이 논의할 항목
- 사람 검토가 반드시 필요한 출력은 어디까지인가
- 예측 정확도와 설명 품질 중 무엇을 더 우선할 것인가
- 채널별 설명의 최소 근거 기준을 어떻게 둘 것인가
- 활용 가능한 응답의 판정 기준을 어떻게 정할 것인가
