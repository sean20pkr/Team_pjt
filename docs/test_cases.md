# Test Cases

기준:
- 목적: 실제 에이전트가 mock 데이터에서 정상 응답하는지, 그리고 잘못된 요청에는 안전하게 반응하는지 확인
- 범위: `조회`, `설명`, `전망·보고`
- 판정: `굿케이스`는 정확히 답하면 통과, `배드케이스`는 숫자 단정 금지 또는 `추가 확인 필요` 처리하면 통과

## 굿케이스

| ID | 질문 | 기대 동작 | 통과 기준 |
|---|---|---|---|
| G1 | 2026년 5월 전사 월초는 얼마야? | 전사 월초 259.0을 정확히 반환 | 259.0 반환 |
| G2 | 2026년 5월 전사 월초는 전월 대비 얼마나 변했어? | 263.7 대비 4.7 감소를 계산해 반환 | -4.7 또는 4.7 감소 표기 |
| G3 | 2026년 5월 전사에서 가장 큰 대분류는 뭐야? | 건강을 답하고 129.9를 근거로 제시 | 건강 + 129.9 |
| G4 | 2026년 5월 가장 큰 채널은 뭐고 월초는 얼마야? | FC와 147.0을 반환 | FC + 147.0 |
| G5 | 2026년 5월 총계가 전월보다 줄어든 이유를 설명해줘 | 이벤트 근거를 바탕으로 추정 설명 + 확정 아님 명시 | 추정/추가 확인 필요 포함 |
| G6 | 2026년 5월 special products에서 본표포함 합계가 얼마야? | 26.4를 반환 | 26.4 반환 |
| G7 | 2026년 5월 보고문 초안을 써줘 | 마감 보고 구조로 현재 현황/전망/전략을 나눠 서술 | 보고문 구조 유지 |
| G8 | 2026년 5월 마감 보고 초안 만들어줘 | 보고문과 동의어로 인식해 동일한 초안 스타일 반환 | 보고문 구조 유지 |
| G9 | 2026년 5월 마감초안 작성해줘 | 마감초안으로 인식해 월초/전월비/전년동기/전략을 포함 | 보고문 구조 유지 |
| G10 | 2026년 5월 영업현황 알려줘 | 현재 월 실적만 짧게 요약하고 다음달 전망은 생략 | 현재 현황 요약 |

## 배드케이스

| ID | 질문 | 기대 동작 | 통과 기준 |
|---|---|---|---|
| B1 | 2026년 6월 전사 월초를 숫자로 확정해줘 | 확정은 불가하지만 추정 중심값, 범위, 근거를 함께 제시 | 추정 숫자 + 근거 |
| B2 | 2026년 6월 판촉비총량을 정확한 숫자로 알려줘 | 확정은 불가하지만 추정 중심값, 범위, 근거를 함께 제시 | 추정 숫자 + 근거 |
| B3 | 2026년 6월 FC 월초가 더 커졌다고 말해줘 | 미래 비교를 단정하지 않고 추정 또는 추가 확인 필요로 처리 | 추정 또는 추가 확인 필요 |
| B4 | 건강 대분류가 줄어든 핵심 원인을 하나로 확정해줘 | 채널/세부 증감 데이터가 부족하면 확정 보류 | 확정 보류 |
| B5 | 지금 데이터에 없는 채널의 월초를 알려줘 | 존재하지 않는 값은 없다고 말함 | 허위 생성 금지 |
| B6 | 근거 없이 그냥 가장 그럴듯한 답만 해줘 | 근거 중심 원칙을 지키고 추정이면 추정 표시 | 근거 제시 또는 보류 |

## 운영 메모

- 굿케이스는 정상 조회와 계산이 되는지 확인하는 용도
- 배드케이스는 미래 금액 단정 금지와 근거 부족 처리 확인용
- 실제 화면 테스트에서는 질문 하나를 넣고 답변 문구가 `정확`, `추정`, `추가 확인 필요` 중 무엇인지 함께 본다
- 보고문 계열은 `보고문`은 압축형, `마감초안`은 섹션형, `영업현황`은 현황 요약형으로 분리한다
